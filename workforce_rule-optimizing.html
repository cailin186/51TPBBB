If we add/edit/On/Off a workforce rule here, this rule will only take effect to the training(workforce) which meet the following conditions：

• The training(workforce) has no leads booked yet

• The start time of the training(workforce) is later than the saved time of this rule

We can no longer change the SERVICE TYPE if we saved the rule successfully.

(For the service type is the most important and only object to a workforce/training, it will influence lots of function if we change it after we saved it.)


When we save the rule, If session in this rule is just the same as session in other rule, the system will highlight this session in this rule, and we should delete it so as to save this rule successfully.

("Just the same" means all the workforce type/Booking rule/Cycle/Duration/session are the same.)

• Workforce ID:系统为这条规则自动生成的系统ID


• Workforce Name:这条规则的自定义名称，必填

• Service Type:单选，必填，默认选中“select”，再次编辑时不可更改。选项包括Pre-screen/Tech Check/PSO/NTT/Demo/ OB/Completion/NTT-Bridge/Demo-Bridge。（DEMO BRIDGE选项待定）


• CBT Type:必填，"Unlimited"与“Specific Type”为单选按钮组，默认选中"Unlimited"，当且仅当选中“Specific Type”时才展示下方多选按钮组。多选按钮组选项为系统中当前所有的CBT NAME与HBT。


• Training Type：选项与默认选项如图。当且仅当SERVICE中选中NTT/NTT-Bridge时才展示此选项。


• Source Type:选项与默认选项如图。

• 编辑已有规则后，变更内容仅对符合以下条件的排班SESSION生效：

• 1.此SESSION尚无LEADS预约

• 2.此SESSION的开始时间晚于这条规则变更后的保存时间


• 保存规则时的校验：

• 1.若必填项为空(包括多选项选中“Specific”时，结果选中项小于1)，则报错：“XXX is required” XXX为必填字段名

• 2.已开启的所有规则中，已存在与当前规则所创建的SESSION条件完全一样(workforce type/Booking rule/Cycle/Duration/SESSION起止时间)的SESSION，则在当前规则中高亮这些被重复的SESSION，并提示：“ Session with the same conditions already exists”

• CYCLE:所配置的SESSION的循环方式。选项和默认选中项如图。“Never”意味着SESSION仅生效一次。循环需要跳过的节假日待定。


• DURATION:所配置SESSION的生效时间。选项和默认选中项如图。“Unlimited”意味着永远生效.当且仅当选中Specific Duration时展示日期时间段编辑框。


• SESSION：排班SESSION。不能为空，至少有一个SESSION。

• 当且仅当CYCLE选择DAILY时，展示DAILITY类型的排班：精确到时分秒的时间段，可增删时间段。

• 当且仅当CYCLE选择WEEKLY时，展示WEEKLY类型的排班：一次可编辑一整周的排班，默认每天均展示一条待编辑的精确到时分秒时间段，可增删时间段，无排班的日期，SESSION可以留空。

• 当且仅当CYCLE选择NEVER时，展示NEVER类型的排班：精确到年月日时分秒的时间段，可增删时间段。
